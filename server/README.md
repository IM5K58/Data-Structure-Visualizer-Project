# 백엔드 서버 (Vierasion Compiler Server) 파일 구조 및 역할 명세서

C++ 코드를 받아서 컴파일, 실행하고 시각화를 위한 메모리 추적(Trace) 정보를 프론트엔드에 전달하는 백엔드 서버입니다.

## 📂 디렉토리 구조 및 파일별 역할

### 🛠 최상위 설정 파일들
*   **`package.json` & `package-lock.json`**
    *   Node.js 프로젝트의 종속성(라이브러리) 목록과 스크립트가 정의된 파일입니다.
    *   `Express`, `TypeScript`, `CORS` 등의 라이브러리 정보가 담겨있습니다.
*   **`tsconfig.json`**
    *   TypeScript 컴파일러 설정 파일입니다.
*   **`docker-compose.yml`**
    *   추후 상용화나 배포 시 안전한 샌드박스 환경(Piston)에서 코드를 실행시키기 위한 Docker 구성 파일입니다. (현재 로컬 테스트 환경에서는 사용하지 않고 직접 g++를 사용합니다.)

### 🚀 `src/` (핵심 소스코드)
서버를 구동하고 API 요청을 처리하는 실제 로직들이 위치한 폴더입니다.

*   **`index.ts`**
    *   **역할:** 서버의 `진입점 (Entry Point)`.
    *   `Express` 웹 서버를 설정하고 실행(포트 3001)시키며, 프론트엔드(`localhost:5173`)와의 통신을 위한 CORS 규칙을 정의합니다.
*   **`routes/compile.ts`**
    *   **역할:** API 라우터 (길잡이).
    *   프론트엔드에서 보내는 `POST /api/compile` 요청을 응대합니다.
    *   코드를 받아서 컴파일러 서비스에 넘기고, 결과(성공 여부, 에러 메시지, 추적 로그)를 규격에 맞춰 JSON 형태로 프론트엔드에 응답합니다.
*   **`services/compiler.ts`**
    *   **역할:** 실제 코드 컴파일 및 실행 대행자.
    *   전달받은 C++ 코드를 임시 폴더에 저장한 뒤, 컴퓨터에 설치된 **`g++` 컴파일러를 직접 호출**하여 바이너리(`main.exe`) 파일을 만들고 실행합니다.
*   **`services/instrumenter.ts`**
    *   **역할:** C++ 코드 자동 계측기 (**이 프로젝트의 가장 핵심적인 스크립트 엔진!**).
    *   사용자가 작성한 평범한 C++ 코드 안에 숨겨진 포인터 할당(`new`), 멤버 변수 접근(`->`), 삭제(`delete`) 등을 정규표현식으로 찾아냅니다.
    *   찾아낸 코드 사이에 시각화를 위한 **상태 추적 함수(`__vt::...`)를 몰래 끼워넣어 변환**하는 역할을 합니다.
*   **`types/index.ts`**
    *   **역할:** 타입 정의 (Type Definitions).
    *   Typescript에서 사용하는 `CompileRequest`, `CompileResponse` 등 통신 규격을 엄격하게 정의하여 오류를 방지합니다.

### 📦 `sandbox/` (C++ 추적 라이브러리)
*   **`__tracer.h`**
    *   **역할:** C++ 런타임 추적 헤더 파일.
    *   `instrumenter.ts`가 삽입한 `__vt::alloc()`, `__vt::set_ptr()` 등의 함수들이 실제로 정의된 C++ 라이브러리입니다.
    *   이 함수들이 실행되면서 `__TRACE__{"type":"ALLOC", ... }` 형태의 비밀 JSON 로그를 출력(stdout)하여 변수의 생명주기를 기록합니다.

### 🗑 기타 임시/테스트 파일들 (삭제 무방)
*   `test_instrumenter.ts` : `instrumenter.ts` 정규표현식을 뜯어고칠 때 썼던 디버깅용 스크립트.
*   `test.cpp`, `out.cpp` 등 : 로컬 `g++` 작동 테스트용 C++ 파일들 파일.
